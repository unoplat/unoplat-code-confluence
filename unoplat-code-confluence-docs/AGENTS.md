## Dependency Guide

- `react`: React is a library for building web and native user interfaces, letting you create interfaces from components that you compose into screens, pages, and apps. React uses JSX to define components and encourages composing small, reusable components into larger UIs. In TypeScript projects, you get React type definitions via @types/react and can type component props and Hooks for safer, editor-assisted development.
- `@tailwindcss/postcss`: Official Tailwind CSS PostCSS plugin used to integrate Tailwind CSS into PostCSS-based build pipelines and frameworks. It is added to your PostCSS configuration as a plugin so Tailwind can generate utilities during CSS processing. It works alongside Tailwind CSS and PostCSS in build tools/frameworks (e.g., Vite, Next.js, Angular) to compile Tailwind’s styles into your output CSS.
- `class-variance-authority`: Class Variance Authority (cva) helps remove the pain of creating class-name variants with traditional CSS, so you can focus on UI development without CSS-in-JS. You define base classes and variant mappings (including compound and default variants) with the cva function to generate class strings for component props. It also provides TypeScript helpers like VariantProps to extract variant types for strongly typed component APIs.
- `react-dom`: The react-dom package contains methods that are only supported for web applications that run in the browser DOM environment and are not supported in React Native. It provides DOM-specific APIs such as portals and synchronous updates, plus resource preloading helpers to speed up web apps. It also exposes client and server entry points (react-dom/client and react-dom/server) for rendering React components in the browser and on the server.
- `postcss`: PostCSS is a tool for transforming CSS with JavaScript. It provides a processor API that runs a pipeline of plugins to parse, transform, and serialize CSS. Plugins can be composed to add features like vendor prefixing, future‑CSS transpilation, modular scoping, or linting through PostCSS-compatible tools.
- `@tanstack/react-router`: TanStack Router is a router for building React and Solid applications, offering fully type-safe APIs and first-class search-params for managing URL state. It provides type-safe navigation, nested and layout routing, file- or code-based route definitions, and built-in route loaders with SWR-style caching and prefetching. It also includes JSON-first search-param state management with schema validation, custom parsing, and middleware for matching/loading.
- `@radix-ui/react-slot`: Utility component that merges its props onto its immediate child, enabling composition patterns like custom asChild APIs. It provides Slot and Slottable components to forward props to a child element while keeping your component’s API flexible. Slot merges props (including event handlers) onto the child and lets the child handler take precedence during composition.
- `lucide-static`: Static assets and utilities for Lucide icons that work without JavaScript frameworks, providing multiple formats like individual SVG files, SVG sprites, icon fonts, and Node.js SVG-string utilities for server-side and static rendering. Provides individual SVG files, an SVG sprite, icon font files, and a JavaScript library exporting SVG strings for flexible use without a framework. Supports use cases such as embedding raw SVGs or sprites in HTML/CSS and importing PascalCase icon strings in Node.js (ESM/CommonJS) for server-side rendering or static sites.
- `@tanstack/react-router-devtools`: Provides dedicated devtools for TanStack Router to visualize its inner workings and aid debugging during development. It exposes React components such as TanStackRouterDevtools and TanStackRouterDevtoolsPanel that connect to a router instance and render an interactive devtools UI. The devtools can be embedded in your app, mounted as a floating panel, and configured with options like initial open state, position, and Shadow DOM targets.
- `lucide-react`: React components for Lucide icons that integrate into React apps; each icon is a fully typed component rendering an optimized inline SVG for vector graphics performance and flexibility. Icons can be imported as React components with full TypeScript support and customized via props such as size, color, stroke width, and other SVG attributes. Tree-shaking includes only the icons you use, and you can use DynamicIcon for name-based rendering when needed.
- `@tanstack/react-start`: Full‑stack React framework powered by TanStack Router that provides SSR, streaming, server functions, API routes, bundling and more, ready to deploy to common hosting providers. It ships with an enterprise‑grade, fully type‑safe routing system and type‑safe full‑stack APIs built on TanStack Router. It supports full‑document SSR and streaming, plus server functions/RPCs and related server‑side capabilities while remaining client‑side friendly.
- `fumadocs-ui`: The default Fumadocs UI provides a beautifully designed theme for documentation sites, bundled with many interactive components and layouts. It delivers a ready‑made documentation UI with interactive components and layouts that reduce maintenance while receiving regular improvements. It can be customized or replaced via the Fumadocs CLI, and it also supports swapping to the Base UI package if you want a headless variant.
- `fumadocs-core`: Headless core library of Fumadocs that provides server-side functions and headless components for building documentation in React.js frameworks like Next.js. It includes built-in document search (e.g., Orama or Algolia) along with breadcrumb and table‑of‑contents components to help structure docs navigation. It also ships remark/rehype plugins and utilities so you can integrate content sources and build custom docs experiences without the full UI layer.
- `clsx`: A tiny utility for constructing className strings conditionally, designed as a faster and smaller drop‑in replacement for the classnames module. It accepts any number of arguments and merges strings, arrays, and objects into a single className string while discarding falsey values. It supports nested arrays and object syntax so you can conditionally include classes without manual string concatenation.
- `tailwind-merge`: Utility function to efficiently merge Tailwind CSS classes in JavaScript without style conflicts. It resolves conflicts between Tailwind class groups so the last conflicting class wins while preserving non-conflicting classes. It supports modern Tailwind versions, works in modern browsers and maintained Node environments, and ships with TypeScript types for typed usage.
- `feed`: TypeScript‑built RSS 2.0, Atom 1.0, and JSON Feed 1.0 generator for Node.js, aimed at making content syndication simple and intuitive. It provides a strongly typed Feed API to programmatically construct feed metadata and items, then output RSS, Atom, or JSON Feed formats. The library is tested for each format and is intended for developer‑friendly generation of syndication feeds in Node.js projects.
- `vite`: Vite is a blazing fast frontend build tool powering the next generation of web applications. It provides instant server start with on‑demand native ESM file serving and blazing fast dependency pre‑bundling, plus lightning‑fast hot module replacement during development. It also delivers rich out‑of‑the‑box features like TypeScript/JSX/CSS support and an optimized production build with advanced tree‑shaking and minification.
- `@tanstack/start-static-server-functions`: Provides static server functions for TanStack Start that run at build time and are cached as static assets during prerendering/static generation. It exposes staticFunctionMiddleware to enable this static mode in createServerFn workflows. You apply staticFunctionMiddleware to a server function so it executes during build-time prerendering and its result is stored as a static JSON asset keyed by the function ID and input. At runtime, the prerendered HTML hydrates the embedded data and subsequent client invocations fetch the cached static JSON instead of executing the function on the server.
- `fumadocs-mdx`: Fumadocs MDX is the official content source of Fumadocs that transforms content into type-safe data, similar to Content Collections. It is a content processing layer for React frameworks for handling MDX/Markdown content like documentation and blog posts. It compiles Markdown/MDX collections into type-safe data and React Server Components with useful exports such as frontmatter and table of contents. It provides configurable MDX processing via presets/plugins and integrates with React frameworks (e.g., Next.js or Vite) through its plugins and configuration files.
- `@radix-ui/react-separator`: Visually or semantically separates content. It provides a Separator.Root component that supports horizontal and vertical orientations for dividing content. It supports decorative separators and adheres to the ARIA separator role requirements for accessibility.

## Business Logic Domain

This codebase models documentation site metadata for release communications and UI announcements. The `BannerConfig` defines configurable marketing or alert banners with identifiers, messages, links, and visual variants. The `ChangelogEntry` represents structured release notes with versioning, dates, and optional GitHub release references, keeping documentation-site messaging and changelog publication consistent.

Key data models:
- `src/lib/banner-config.ts`: Marketing/alert banner configuration (identifiers, messages, link metadata, and visual variants).
- `src/lib/changelog-utils.ts`: Structured changelog entries (versioning, dates, summaries, and optional GitHub release references).

Reference details: see `business_logic_references.md`.
