<Callout type="info" title="Our Mission">
  Unoplat Code Confluence is a **context engine** for application development. It turns repository evidence into concise, traceable guidance so developers and coding agents can spend less time rediscovering how an existing codebase works.
</Callout>

## The Solution

### Four complementary outputs

Unoplat Code Confluence generates `AGENTS.md` guidance and focused companion artifacts for four key areas:

<Cards>
  <Card icon={<Settings />} title="Engineering Workflow">
    Canonical **install**, **build**, **dev**, **test**, **lint**, and **type-check** commands grounded in repository configuration.
  </Card>
  <Card icon={<Package />} title="Dependency Guide">
    A current dependency inventory with concise package-purpose context in `dependencies_overview.md`.
  </Card>
  <Card icon={<Brain />} title="Business Domain">
    A concise domain summary backed by detected data models and line-level references in `business_domain_references.md`.
  </Card>
  <Card icon={<GitBranch />} title="Repository Architecture">
    A validated repository-level diagram grounded in fresh, source-anchored application boundaries from `app_interfaces.md`.
  </Card>
</Cards>

Architecture evidence begins with framework and library operations classified by the [published DIDS capability enum](/docs/contribution/custom-framework-schema#published-capability-enum). Those detections are normalized into inbound, outbound, bidirectional, or internal constructs with source anchors in `app_interfaces.md`; architecture generation then works from the fresh interface evidence produced by the current run.

See [How it works](/docs/introduction/how-it-works) for the complete evidence pipeline and the generated artifacts behind each output.

### Core Principles

<Cards>
  <Card icon={<Target />} title="Structured Discovery">
    Tree-sitter analysis and [framework-aware definitions](/docs/contribution/custom-framework-schema) identify supported language and library patterns before synthesis.
  </Card>
  <Card icon={<Layers />} title="Evidence-Linked Context">
    Detected dependencies, application boundaries, and domain models retain source references where the generated artifact supports them.
  </Card>
  <Card icon={<Shield />} title="Auditable Workflows">
    Orchestrated generation provides granular progress and separates ownership of generated artifacts.
  </Card>
  <Card icon={<Zap />} title="Focused Processing">
    Targeted source spans, fresh interface anchors, incremental dependency reuse, and no-change checks reduce unnecessary repeated work.
  </Card>
</Cards>
