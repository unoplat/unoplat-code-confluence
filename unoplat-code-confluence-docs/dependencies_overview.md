# Dependencies Overview

- `@tanstack/start-static-server-functions`: Provides middleware to enable TanStack Start server functions to run at build time and be cached as static assets for prerendering/static generation. It exports staticFunctionMiddleware for createServerFn so server functions can execute during prerendering and cache their results as static JSON assets. At runtime, the prerendered HTML hydrates embedded data and future client calls fetch the cached static JSON output instead of executing on the server.
- `motion`: Motion is a production-grade, free and open-source animation library for the web that helps create high-performance animations for React, JavaScript, and Vue. It provides simple, declarative APIs like the React <motion /> components and JavaScript animate() for building smooth UI animations and gestures. Its hybrid engine combines JavaScript flexibility with native browser animation performance, supporting advanced effects like layout, scroll, and gesture-driven animations.
- `react-dom`: The react-dom package contains methods supported only for web applications that run in a browser DOM environment. It serves as the entry point to the DOM and server renderers for React, intended to be paired with the core react package. It provides client-side APIs like createRoot for rendering React components into the browser DOM and other web-only utilities such as portals and synchronous flushing. It also includes server-side rendering APIs (e.g., in react-dom/server) for generating HTML streams for React apps.
- `class-variance-authority`: Class Variance Authority aims to remove the pain of manually matching classes to props and adding types when creating variants, letting you focus on UI development. It provides a cva function to build class-producing component functions with base classes, variant schemas, compound variants, and default variants. It also includes TypeScript helpers like VariantProps to extract strongly typed variant props for your components.
- `@tanstack/react-router-devtools`: TanStack Router provides dedicated devtools to visualize the router's inner workings and aid debugging of React route state and behavior. These devtools are delivered as a separate package for React applications. It renders interactive DevTools UI for TanStack Router, including floating, fixed, or embedded panels and can connect automatically when placed inside the RouterProvider or via an explicit router prop. It also supports configuration options such as initial open state, panel/toggle props, position, and Shadow DOM targeting for styling isolation.
- `tailwind-merge`: Utility function to efficiently merge Tailwind CSS classes in JS without style conflicts. It merges multiple Tailwind class strings and removes conflicting classes so the last intended utilities win while non-conflicting classes are preserved. It supports modern Tailwind versions, works in modern browsers and Node.js, and ships with TypeScript typings for safe use in TS projects.
- `clsx`: A tiny utility for constructing className strings conditionally, and a faster, smaller drop-in replacement for the classnames module. It builds a single space-separated class string from mixed inputs such as strings, objects, arrays, and booleans, discarding falsey values. It is distributed in multiple module formats (ESM, CommonJS, UMD) and supports a string-only lite variant for minimal class-building needs.
- `@radix-ui/react-slot`: Merges its props onto its immediate child and can be used to support an asChild prop. Provides Slot.Root to forward and merge props into a single immediate child for building polymorphic components. Includes Slot.Slottable to target the correct child when multiple children are present, enabling a custom asChild API.
- `@tanstack/react-router`: TanStack Router is a router for building React and Solid applications, offering fully type-safe APIs and first-class search-params for URL state management. It is designed for client-side and full-stack React apps with seamless ecosystem integration and built-in features like route loaders and caching. It provides type-safe navigation, nested/layout routes, and schema-validated path/search params to manage URL state confidently. It also includes route loaders with caching, automatic prefetching, and support for file-based or code-based routing to streamline app architecture.
- `react`: React is a library for web and native user interfaces that lets you build UIs from components. It enables you to create components and compose them into screens, pages, and apps for building user interfaces. It provides core APIs and built-in features like components and Hooks to add state and other React capabilities inside your components.
- `@radix-ui/react-separator`: Provides the Separator primitive that visually or semantically separates content in React interfaces, as part of Radix Primitives. It renders a Separator.Root component that can be oriented horizontally or vertically for layout dividers. It supports accessibility via the separator role and offers props like orientation, decorative, and asChild for customization and composition.
- `lucide-react`: React components for Lucide icons that integrate seamlessly into React applications, with each icon as a fully typed React component rendering an optimized inline SVG. Import icons as React components with full TypeScript support and pass props to customize size, color, stroke width, and other SVG attributes. Lucide React is tree-shakable, so only the icons you import are included in your bundle and it supports dynamic icon components when needed.
- `postcss`: PostCSS is a tool for transforming CSS with JavaScript, enabling processing of styles via plugins and APIs. It provides an API that turns CSS into an AST so plugins can analyze and modify rules. PostCSS processes CSS by running a pipeline of plugins that can lint, transform, and optimize styles, such as adding vendor prefixes or enabling future CSS syntax. It exposes a JavaScript API that parses CSS into an AST and lets plugins or tooling read and rewrite rules before output is generated.
- `lucide-static`: Static assets and utilities for Lucide icons that work without JavaScript frameworks, providing multiple formats such as individual SVG files, SVG sprites, icon fonts, and Node.js utilities for SSR and static site generation. It includes individual SVG files, an SVG sprite, icon font files, and a JavaScript library exporting SVG strings so you can choose the format that fits your build. It's intended for cases like embedding raw SVGs or sprites, using icon fonts or CSS background images, or importing SVG strings in Node.js without relying on a framework.
- `fumadocs-mdx`: Fumadocs MDX is the official content source for Fumadocs, transforming content into type-safe data for React frameworks. It acts as a content processing layer rather than a full CMS, suitable for handling docs, blog posts, and other content collections. It defines collections in a config file and compiles Markdown/MDX into structured outputs like frontmatter, table of contents, and structured data for use in your app. It provides configurable MDX processing with built-in defaults (e.g., syntax highlighting and TOC extraction) and allows customization via global or per-collection options.
- `@tanstack/react-start`: TanStack React Start is a full-stack React framework powered by TanStack Router and Vite, providing SSR, streaming, server functions, API routes, and bundling for deployment to your hosting provider. It provides enterprise-grade, fully type-safe routing built on TanStack Router and extends that type safety to full-stack APIs. It supports server rendering and streaming with server functions/RPCs while remaining client-side-first and server-capable for rich interactive apps.
- `fumadocs-core`: Fumadocs Core offers server-side functions and headless components for building documentation in the App Router. It provides search, navigation components (breadcrumb, sidebar, TOC), remark/rehype plugins, and utilities for flexible docs building. It supplies headless building blocks such as search integration and navigation/TOC components that you compose into your own UI. It also includes content-processing plugins and utility APIs to power documentation sites without requiring the Fumadocs UI layer.
- `feed`: A Node.js library that generates RSS 2.0, Atom 1.0, and JSON Feed 1.0 syndication feeds, aiming to make content syndication simple and intuitive. It provides a typed Feed class for defining feed metadata and adding items, contributors, categories, and other fields before rendering output. It can emit RSS 2.0, Atom 1.0, or JSON Feed 1.0 strings from the same feed data.
- `vite`: Vite is a next-generation frontend build tool that provides a fast development server and optimized build for modern web applications. It serves source files on demand over native ESM with lightning-fast hot module replacement for quick feedback during development. It offers out-of-the-box support for TypeScript/JSX/CSS plus a Rollup-based production build and a universal plugin system.
- `fumadocs-ui`: Fumadocs UI is the default theme of Fumadocs, offering a beautifully designed documentation site UI with interactive components and layouts. It provides a ready-made UI theme and components for building documentation sites within the Fumadocs ecosystem. It integrates with Fumadocs Core and related tooling to deliver layouts and interactive docs features with low maintenance.
- `@tailwindcss/postcss`: Official Tailwind CSS PostCSS plugin used to integrate Tailwind CSS with PostCSS-based build pipelines and frameworks. It registers Tailwind CSS as a PostCSS plugin so your build can generate the utility classes into your compiled CSS. It works alongside PostCSS tooling to process your project's CSS imports and produce the final stylesheet.
