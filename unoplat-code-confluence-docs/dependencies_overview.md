# Dependencies Overview

- **Package management**: Use `bun` as the package manager for this codebase.
- **@tailwindcss/postcss**: Purpose: PostCSS plugin for Tailwind CSS v4. Add it to your PostCSS configuration to integrate Tailwind into builds like Next.js, Angular, and other PostCSS-based setups. Source: https://tailwindcss.com/docs/installation/using-postcss
- **@tanstack/react-router**: Purpose: A powerful, fully type-safe router for React applications, with first-class search-parameter APIs and support for client-side and full-stack routing. It provides nested/layout routes, route loaders with caching, automatic prefetching, and TypeScript-first navigation.
- **@tanstack/react-router-devtools**: Purpose: Dedicated devtools package for TanStack Router that helps visualize the router’s internal workings and debug routing issues. It provides components such as TanStackRouterDevtools and TanStackRouterDevtoolsPanel for floating, fixed, or embedded use.
- **@tanstack/react-start**: Purpose: A full-stack React framework powered by TanStack Router. It provides full-document SSR, streaming, server functions, bundling, and deployment to Vite-compatible hosting providers.
- **@tanstack/react-table**: Purpose: React adapter for TanStack Table's headless core table logic. It provides the `useReactTable` hook, React-style state management, and rendering support for cell/header/footer templates.
- **@tanstack/start-static-server-functions**: Purpose: Experimental TanStack Start middleware for making server functions static. It runs the function during build-time prerendering, caches the result as a static asset, and hydrates/serves that cached data at runtime.
- **class-variance-authority**: Purpose: A tiny TypeScript utility for building type-safe, variant-driven CSS class name components. It lets you declare variants, compound variants, and defaults declaratively, and works anywhere CSS class names do.
- **clsx**: Purpose: A tiny utility for constructing `className` strings conditionally. It also serves as a smaller drop-in replacement for the `classnames` module.
- **cmdk**: Purpose: A command menu React component that can also be used as an accessible combobox. It renders items, filters and sorts them automatically, and offers a fully composable API for building command palettes.
- **date-fns**: Purpose: Modern JavaScript date utility library for manipulating dates in browser and Node.js. Modular, immutable, and fully TypeScript with a large set of date helpers.
- **feed**: Purpose: Creates RSS 2.0, Atom 1.0, and JSON Feed 1.0 feeds for Node.js apps. The library is TypeScript-based and aims to make content syndication simple and intuitive.
- **fumadocs-core**: Purpose: A headless Fumadocs library that provides server-side functions and components for building docs sites in React frameworks like Next.js. Includes document search, breadcrumb/sidebar/TOC components, remark/rehype plugins, and other utilities.
- **fumadocs-mdx**: Purpose: The official content source of Fumadocs. It transforms Markdown and MDX content into type-safe data for React/Next.js apps, with collections, metadata, and built-in properties like table of contents and frontmatter.
- **fumadocs-ui**: Purpose: The default Fumadocs theme for documentation sites, providing a beautifully designed UI with interactive components and layouts. It supports Radix UI by default, with Base UI available as an alternative component library.
- **lucide-react**: Purpose: React component library for Lucide icons. Each icon is a standalone, tree-shakable inline SVG component with props for size, color, stroke width, and built-in TypeScript support.
- **lucide-static**: Purpose: Provides Lucide icons as static assets for web applications: raw SVG files, a JavaScript library of SVG strings, icon fonts, and an SVG sprite. Best suited for cases where you need standalone assets rather than framework-specific icon components.
- **motion**: Purpose: Motion is an animation library for JavaScript and TypeScript. It provides a simple `animate()` API for animating HTML, SVG, and JavaScript objects, plus motion values and scroll/gesture utilities.
- **postcss**: Purpose: A tool for transforming CSS with JavaScript. It parses CSS into an AST and lets plugins analyze and modify styles, add vendor prefixes, lint, and transpile future CSS syntax.
- **Radix UI React Primitives**: Purpose: A low-level, unstyled React UI component library for building accessible design systems and web apps. Includes primitives like Separator for visually or semantically separating content and Slot for merging props onto an immediate child to support `asChild` composition.
- **radix-ui**: Purpose: An open-source UI component library for building high-quality, accessible design systems and web apps. It ships unstyled, fully typed primitives you can customize and adopt incrementally.
- **react**: Purpose: React is a JavaScript library for building user interfaces with reusable components. In TypeScript projects, it supports JSX and is typically paired with @types/react and @types/react-dom for type definitions. (https://react.dev/learn/describing-the-ui, https://react.dev/learn/typescript)
- **react-day-picker**: Purpose: A customizable date picker component for React for building calendars, date pickers, and date inputs. Supports single, multiple, and range selections, with styling, localization, and accessibility options.
- **react-dom**: Purpose: The `react-dom` package contains methods that are only supported for web applications running in the browser DOM environment. It also provides client and server entry points for rendering React components.
- **tailwind-merge**: Purpose: Utility function to efficiently merge Tailwind CSS classes in JavaScript without style conflicts; fully typed and provides TypeScript declarations for use in TypeScript projects.
- **vite**: Purpose: Vite is a build tool for modern web projects that provides a fast development server with native ESM/HMR and a production build command for optimized static assets. It is extensible through plugins and JavaScript APIs, with full typing support.
