# Dependencies Overview

- **Package management**: Use `bun` as the package manager for this codebase.
- **@tailwindcss/postcss**: Purpose: PostCSS plugin for integrating Tailwind CSS into your build pipeline. Tailwind docs recommend adding `@tailwindcss/postcss` to `postcss.config.mjs` and importing `tailwindcss` in your CSS.
- **@tanstack/react-router**: Purpose: TanStack Router is a router for building React applications with 100% inferred TypeScript support, typesafe navigation, nested routing, and search-param handling.
- **@tanstack/react-router-devtools**: Purpose: TanStack Router’s dedicated devtools package for React apps. It helps visualize router internals for debugging and provides floating, fixed, and embedded devtools panels.
- **@tanstack/react-start**: Purpose: Full-stack React framework powered by TanStack Router and Vite, providing full-document SSR, streaming, server functions, and type-safe full-stack APIs. Sources: [TanStack Start Overview](https://tanstack.com/start/v0/docs/framework/react/overview), [TanStack Start Docs Home](https://tanstack.com/start/latest)
- **@tanstack/react-table**: Purpose: React adapter for TanStack Table’s core table logic. It provides `useReactTable` plus React-style state handling, types, and cell/header/footer rendering support for headless tables.
- **@tanstack/start-static-server-functions**: Purpose: Experimental TanStack Start middleware for static server functions. It provides `staticFunctionMiddleware`, which runs server functions at build time and caches their results as static assets during prerendering/static generation.
- **class-variance-authority**: Purpose: A utility for building type-safe, variant-based class name generators for UI components in TypeScript. It also exposes `VariantProps` to infer variant prop types from `cva` definitions.
- **clsx**: Purpose: A tiny utility for conditionally constructing `className` strings. It accepts strings, arrays, objects, and other values, discarding falsey inputs; `clsx/lite` is a smaller string-only variant.
- **cmdk**: Purpose: Fast, composable, unstyled command menu for React. It can also be used as an accessible combobox and automatically filters and sorts items.
- **date-fns**: Purpose: A modern JavaScript date utility library for manipulating dates in browsers and Node.js. It is modular, immutable, and 100% TypeScript with handcrafted types.
- **feed**: Purpose: RSS 2.0, JSON Feed 1.0, and Atom 1.0 generator for Node.js. It helps you quickly generate content syndication feeds in TypeScript/JavaScript with typed feed and item metadata.
- **fumadocs-core**: Purpose: The headless library for Fumadocs. It provides server-side functions and headless components for building documentation sites on React frameworks like Next.js, including search, breadcrumb/sidebar/TOC components, remark/rehype plugins, and utilities.
- **fumadocs-mdx**: Purpose: Official Fumadocs content source for transforming Markdown/MDX and YAML/JSON collections into type-safe data for React frameworks. It also provides built-in outputs like frontmatter, table of contents, and structured data.
- **fumadocs-ui**: Purpose: The default theme of Fumadocs for documentation sites. It bundles interactive components and layouts for docs sites, with regular UI improvements.
- **lucide-react**: Purpose: A Lucide icon library package for React applications. It provides tree-shakable SVG icons with customization options for color, size, stroke width, and more.
- **lucide-static**: Purpose: Static assets and utilities for Lucide icons that work without JavaScript frameworks. Provides individual SVG files, SVG sprites, icon fonts, and a JavaScript library exporting SVG strings for Node.js/SSR.
- **motion**: Purpose: Motion is an animation library that's easy to start and fun to master. It uses a hybrid engine to animate HTML/CSS, SVG, and WebGL, and is installed via the "motion" package.
- **postcss**: Purpose: PostCSS is a tool for transforming CSS with JavaScript plugins. It can add vendor prefixes, transpile future CSS syntax, lint styles, and more; the package includes built-in TypeScript declarations.
- **Radix UI React Primitives**: Purpose: A low-level, unstyled React UI component library for building accessible design systems and web apps. The Separator primitive visually or semantically separates content, and the Slot utility merges props onto its immediate child to support `asChild` composition.
- **radix-ui**: Purpose: An open-source, low-level UI component library for building accessible design systems and web apps. It ships unstyled, fully-typed primitives with built-in accessibility, keyboard navigation, and tree-shakeable incremental adoption.
- **react**: Purpose: React is a JavaScript library for rendering user interfaces with reusable components and JSX. The official docs also include TypeScript guidance and typings support via `@types/react` and `@types/react-dom`.
- **react-day-picker**: Purpose: A React component for creating date pickers, calendars, and date inputs for web applications. It supports single, multiple, range, and custom selections, and is written in TypeScript.
- **react-dom**: Purpose: Provides React DOM APIs for web applications running in the browser DOM environment. Includes client entry points for rendering/hydrating roots and server entry points for rendering React on the server.
- **tailwind-merge**: Purpose: Utility function to efficiently merge Tailwind CSS classes in JavaScript/TypeScript without style conflicts. It resolves conflicting utility classes by keeping the last applicable class and is fully typed.
- **vite**: Purpose: Vite is a blazing fast frontend build tool for modern web applications. It provides a dev server over native ES modules, fast HMR, and an optimized production build with full TypeScript typing support.
