Typography & Component Integration

When implementing typography within components, follow these general principles:

  • Maintain consistent text sizes within component categories (all buttons should use the same text size)
  • Use font weight rather than size to create hierarchy within similar elements
  • For data-dense interfaces like tables, use more compact typography (text-xs, text-sm)
  • Ensure proper spacing between text elements (labels and fields, headers and content)
  • Apply semantic colors based on the color system for interactive states
  • Maintain readable line length for multiline text (65-75 characters maximum)

Accessibility

Accessibility is a core principle. Our foundations ensure that the UI is usable by everyone, regardless of ability.

Color Contrast

We adhere to WCAG 2.1 AA guidelines for color contrast. All text and essential UI elements must have a minimum contrast ratio:

  • 4.5:1 for normal text (less than 18pt or 14pt bold)
  • 3:1 for large text (18pt or larger, or 14pt bold or larger)
  • 3:1 for graphical elements and user interface components

Keyboard Navigation

All interactive elements must be navigable and operable using only a keyboard.

  • Logical focus order following the visual layout.
  • Visible focus indicators (outlines) for all focusable elements. Use focus:ring utilities.
  • Standard key interactions (Enter/Space for buttons, Arrow keys for menus, etc.).

Screen Reader Support

Ensure content is understandable and navigable by screen readers.

  • Use semantic HTML elements (<nav>, <button>, <main>, etc.).
  • Provide descriptive alt text for all meaningful images.
  • Use ARIA attributes when necessary to enhance semantics (e.g., aria-label, aria-hidden).
  • Ensure form elements have associated labels.
