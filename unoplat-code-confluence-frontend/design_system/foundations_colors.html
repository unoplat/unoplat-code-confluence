Example Usage in CSS

.button-primary {
  background-color: var(--color-primary);

Best Practices for Interactive States

  • Maintain a minimum contrast ratio of 4.5:1 for all interactive elements to ensure accessibility.
  • Use consistent color patterns across similar UI elements to build user familiarity.
  • Always pair color changes with other visual indicators (icons, shapes, text) for users with color vision deficiencies.
  • Avoid using color alone to convey information or state changes.
  • Ensure disabled states are clearly distinguishable but not too prominent in the interface.

Data Visualization Best Practices

  • Choose appropriate color scales based on the type of data (categorical, sequential, or diverging).
  • Limit categorical palettes to 6-8 colors maximum for clear differentiation.
  • Ensure sufficient contrast between adjacent colors in sequential and diverging palettes.
  • Use consistent colors for the same data categories across different visualizations.
  • Consider color blindness by ensuring visualizations are distinguishable without relying solely on color.
  • Add patterns or textures for additional differentiation when necessary.

Semantic Color Usage

Using colors semantically provides clear meaning and context to UI elements. Here are the guidelines for applying semantic colors:

Color Token Meaning Usage Examples
--color-primary 		Primary action, active state, brand Main call-to-action buttons, active navigation links, form field focus outlines, branding elements.
--color-success 		Positive feedback, completion Success messages, validation success indicators, completed status badges.
--color-warning 		Caution, non-critical alert Warning messages, attention-required badges, potentially risky actions (non-destructive).
--color-error 		Negative feedback, critical alert Error messages, validation failures, destructive action confirmations, failed status indicators.
--color-info 		Neutral information Informational alerts, tips, neutral status indicators, non-interactive highlights.
--color-neutral 		Supporting elements, secondary content Secondary text, disabled states (often combined with reduced opacity), dividers, borders, subtle backgrounds.
