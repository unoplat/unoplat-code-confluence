Form Structure

Forms are organized in consistent layouts to improve usability and understanding. This section provides guidelines for structuring forms, including nested and multi-step patterns.

Basic Form Structure

Guidelines
  • Use consistent vertical spacing: space-y-6 between form groups
  • Place labels above inputs with text-sm font-medium text-gray-700 mb-1
  • Set button positioning with flex justify-end or flex justify-between for multiple buttons

Form Validation

Form validation provides feedback to users about the correctness of their input. This section covers validation patterns and visual indicators to guide users through form completion.

Inline Validation

Email is valid

Please enter a valid email address

Inline Validation Styling
  • Valid input: border-green-500 focus:border-green-500 focus:ring-green-500
  • Invalid input: border-red-500 focus:border-red-500 focus:ring-red-500
  • Validation message: mt-1 text-sm text-red-600
  • Include validation icons for clear visual indication

Validation Timing

1

On Blur Validation

Validate when a field loses focus. This gives users time to complete their input before showing errors.

2

Immediate Validation

Show success states as soon as valid input is detected. For errors, prefer to wait until blur to avoid interrupting the user.

3

Submit Validation

Always validate upon form submission and prevent submission if errors exist. Highlight all fields with errors.

Form-Level Validation

  • Your password must be at least 8 characters
  • Your password must include at least one number

Password must be at least 8 characters

Form-Level Validation Guidelines
  • Error Summary: bg-red-50 p-4 with text-red-800 heading
  • List all errors in a summary at the top of the form
  • Link error messages to corresponding fields when possible
  • Maintain individual field error states and messages

Authentication Modal Guidelines
  • Clear instructions about required credentials
  • Secure input fields (password type) for tokens
  • Information about token security and storage
  • Option to remember credentials when appropriate
  • Cancel and authenticate actions clearly labeled

Confirmation Dialog

Delete Repository

Are you sure you want to delete this repository? This action cannot be undone.

Confirmation Dialog Guidelines
  • Use clear, direct language about the action and its consequences
  • For destructive actions, use red color and warning icons
  • Provide "Cancel" option as the secondary action
  • Primary action button should reflect the action (e.g., "Delete")
  • Keep dialog focused and concise

Notification Components

Notification components provide feedback to users about system events, updates, and status changes. These components should be informative yet unobtrusive.

Toast Notification Guidelines
  • Container: max-w-md w-full bg-white shadow-lg rounded-lg pointer-events-auto
  • Color indicator: border-l-4 border-{color}-500
  • Include icon, title, message, and close button
  • For actionable notifications, include clear buttons
  • Auto-dismiss non-critical notifications after 5-7 seconds
  • Position in top-right or bottom-right of viewport
