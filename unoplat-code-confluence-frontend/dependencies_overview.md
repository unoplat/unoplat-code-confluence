# Dependencies Overview

- **Package management**: Use `bun` as the package manager for this codebase.
- **@hugeicons/core-free-icons**: Purpose: Free Hugeicons icon pack for TypeScript/JavaScript apps, with 5,100+ icons in one style: Stroke Rounded. It’s the free set used alongside Hugeicons framework packages such as `@hugeicons/react`.
- **@hugeicons/react**: Purpose: React component package for Hugeicons that exposes a single `HugeiconsIcon` component to render icons imported from Hugeicons packages. Use it with the free or Pro icon packages in React apps to display scalable SVG icons.
- **@mdxeditor/editor**: Purpose: MDXEditor is an open-source React component for authoring markdown documents in a WYSIWYG editor. The package exports the MDXEditor component plus plugins and UI primitives for editing markdown content in React/TypeScript.
- **@playwright/test**: Purpose: Playwright Test is Playwright’s end-to-end test runner for modern web apps. It provides the `test` and `expect` APIs, built-in fixtures and isolation, plus parallel execution and rich tooling for browser tests.
- **@tailwindcss/typography**: Purpose: Tailwind CSS plugin that adds a set of `prose` classes for styling long-form content with sensible typographic defaults. Use it for Markdown, CMS, and other rich-text content you don’t fully control.
- **@tanstack/electric-db-collection**: Purpose: Electric collections provide seamless integration between TanStack DB and ElectricSQL, enabling real-time data synchronization with your Postgres database through Electric’s sync engine. It supports automatic syncing from Postgres shapes, optimistic updates, and customizable mutation handlers.
- **@tanstack/query-db-collection**: Purpose: Provides query-backed collections for TanStack DB, integrating TanStack Query with local collections for automatic synchronization with remote data sources. Supports optimistic updates, customizable persistence handlers, and direct writes to the synced data store.
- **@tanstack/react-db**: Purpose: React bindings for TanStack DB. Provides live-query hooks such as useLiveQuery, useLiveSuspenseQuery, and useLiveInfiniteQuery, plus paced mutations, so components stay automatically in sync with changing data.
- **@tanstack/react-form**: Purpose: A React form state-management library from TanStack Form, with TypeScript-first APIs for creating forms, fields, validation, and form state hooks. It exposes hooks like `useForm`, `useField`, and helpers such as `createFormHook` for building composable forms in React.
- **@tanstack/react-query**: Purpose: A React library for fetching, caching, synchronizing, and updating server state in web applications. It is written in TypeScript and provides strong type inference for queries, mutations, and related helpers.
- **@tanstack/react-query-devtools**: Purpose: TanStack Query’s dedicated React devtools package for inspecting queries and mutations, visualizing cache/state, and debugging React Query behavior. It is installed as a separate package and is intended for development use.
- **@tanstack/react-router**: Purpose: A type-safe router for React applications, with fully inferred TypeScript support. It provides nested routing, route loaders/caching, prefetching, and typesafe search-parameter navigation.
- **@tanstack/react-table**: Purpose: The React adapter for TanStack Table v8; it wraps the core table logic and provides React state management, types, and rendering helpers for table/header/footer templates. It’s a headless library for building powerful tables and datagrids in TS/JS with full control over markup and styles.
- **@tanstack/react-virtual**: Purpose: React adapter for TanStack Virtual, a headless UI utility for virtualizing long lists in React. It provides `useVirtualizer` and `useWindowVirtualizer` hooks that return `Virtualizer` instances for element- or window-based scrolling.
- **@tanstack/zod-adapter**: Purpose: TanStack Router adapter for Zod v3 validation. It provides zodValidator and fallback helpers so route search params can be validated with Zod while preserving correct input/output types.
- **@tanstack/zod-form-adapter**: Purpose: TanStack Form adapter for Zod schemas. It provides `zodValidator` so you can use Zod for form validation in TanStack Form.
- **axios**: Purpose: Promise-based HTTP client for the browser and Node.js. The official docs also include TypeScript examples and API references for TypeScript projects.
- **canvas-confetti**: Purpose: Browser-only confetti animation library for launching animated confetti bursts on a canvas. It supports custom shapes, custom canvases, and a reduced-motion option.
- **class-variance-authority**: Purpose: A tiny TypeScript utility for building type-safe, variant-driven CSS class name components. It lets you declaratively define variants, compound variants, and defaults, with no runtime overhead.
- **clsx**: Purpose: Tiny utility for conditionally constructing `className` strings in JavaScript/TypeScript. Accepts strings, arrays, and objects; falsey values are discarded.
- **cmdk**: Purpose: ⌘K is a command menu React component that can also be used as an accessible combobox. It renders items, then filters and sorts them automatically with a fully composable API.
- **date-fns**: Purpose: A modern JavaScript date utility library for browser and Node.js. It provides a modular, pure, immutable set of date-manipulation helpers and is TypeScript-first.
- **dnd kit**: Purpose: A modern, lightweight drag-and-drop toolkit for building accessible and extensible interfaces in TypeScript. The core package (`@dnd-kit/core`) provides draggable/droppable primitives, while `@dnd-kit/sortable` and `@dnd-kit/utilities` add sortable behaviors and helpers.
- **flyonui**: Purpose: FlyonUI is a free, open-source Tailwind CSS components library with semantic class names and headless JavaScript plugins for building responsive, interactive web apps. It installs as an npm dev dependency and integrates as a Tailwind CSS plugin, with components, blocks, templates, and theming support.
- **json2md**: Purpose: Converts JSON input into Markdown. Supports common Markdown elements like headings, paragraphs, lists, images, tables, code blocks, and links, and can be extended with custom converters.
- **lucide-react**: Purpose: React component library for using Lucide icons in applications. Each icon is a standalone component that renders an optimized inline SVG, with tree-shakable imports and TypeScript support.
- **motion**: Purpose: Motion is a JavaScript animation library for animating HTML/CSS, SVG, WebGL, and JavaScript objects. It includes `animate()`, `scroll`, and motion values for creating performant browser animations.
- **nanoid**: Purpose: A tiny, secure, URL-friendly unique string ID generator for JavaScript/TypeScript. It uses Node.js crypto or the Web Crypto API by default and includes TypeScript support for branded/opaque IDs.
- **next-themes**: Purpose: An abstraction for themes in your React app. Provides a ThemeProvider and useTheme hook for Next.js/React apps with system preference support and no flash on load.
- **nuqs**: Purpose: Type-safe search params state manager for React. Provides hooks and adapters to sync component state with URL query parameters across frameworks like Next.js, Remix, React Router, and TanStack Router.
- **Radix UI React Primitives**: Purpose: An open-source, low-level UI component library for building high-quality, accessible design systems and web apps. Components are unstyled, customizable, and expose a fully typed API for incremental adoption.
- **react**: Purpose: A JavaScript library for building user interfaces from reusable components. React is used to compose components into screens, pages, and apps for web and native platforms. Sources: https://react.dev/ ; https://react.dev/learn/describing-the-ui
- **react-day-picker**: Purpose: React DayPicker is a React component for creating date pickers, calendars, and date inputs for web applications. It is written in TypeScript and supports single, multiple, range, and custom date selections with localization and accessibility features.
- **react-dom**: Purpose: The `react-dom` package provides APIs for web applications that run in the browser DOM environment, including client/server entry points and DOM-specific helpers. It also includes resource preloading utilities and browser-only hooks/components for rendering React on the web.
- **react-icons**: Purpose: React icon library that includes popular icon packs as SVG/React components, using ES6 imports so you can import only the icons your project uses. Provides `IconContext` for global props like color, size, className, and style.
- **react-markdown**: Purpose: React component for rendering Markdown strings safely to React elements. It supports remark/rehype plugins and custom component overrides for markdown-derived elements.
- **shadcn-ui**: Purpose: A set of beautifully designed, accessible components and a code distribution platform for building your own component library. The components are written in TypeScript and can be added directly to your project via the CLI.
- **sonner**: Purpose: An opinionated toast component for React. Usage: mount a `Toaster` and call `toast()` with variants and positioning config.
- **tailwind-merge**: Purpose: Utility for merging Tailwind CSS class strings in JavaScript/TypeScript while resolving conflicting classes. It provides a fully typed `twMerge` function for use in modern browsers and maintained Node versions.
- **vaul**: Purpose: Drawer component for React. Use it to build drawer panels with primitives like `Drawer.Root`, `Drawer.Trigger`, `Drawer.Portal`, `Drawer.Overlay`, and `Drawer.Content`.
- **wicg-inert**: Purpose: Polyfill for the HTML `inert` attribute and property, letting authors mark DOM subtrees as inactive so they don’t receive user interaction events and are hidden from assistive tech. Use `import "wicg-inert"` to load the polyfill in browsers that don’t support `inert` natively.
- **zod**: Purpose: TypeScript-first schema validation library with static type inference. Define schemas to validate untrusted data and parse it into type-safe results. Sources: https://zod.dev/ ; https://zod.dev/packages/zod
- **zustand**: Purpose: Zustand is a small, fast, scalable state management solution for React with a hook-based API. It lets you create stores without providers, reducers, or boilerplate, and pairs well with TypeScript for typed state, actions, and selectors.
