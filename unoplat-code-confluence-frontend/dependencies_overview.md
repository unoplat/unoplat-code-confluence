# Dependencies Overview

- **Package management**: Use `bun` as the package manager for this codebase.
- **@hugeicons/core-free-icons**: Purpose: Free Hugeicons core icon pack for TypeScript apps, with built-in TypeScript declarations. It provides 5,100+ Stroke Rounded icons as assets to use with Hugeicons framework components like @hugeicons/react, @hugeicons/vue, @hugeicons/angular, and @hugeicons/react-native.
- **@hugeicons/react**: Purpose: React wrapper for Hugeicons' icon system. It exposes a single `HugeiconsIcon` component that renders icons imported from Hugeicons core packages, supporting both free and Pro icon sets.
- **@mdxeditor/editor**: Purpose: Open-source React component for authoring markdown documents in a WYSIWYG editor. It accepts and emits markdown strings and is extended through plugins for features like headings, links, tables, code blocks, and JSX.
- **@playwright/test**: Purpose: Playwright Test is Playwright’s end-to-end test framework for modern web apps. It provides the `test` and `expect` APIs, built-in fixtures, and a test runner for browser automation and assertions.
- **@tailwindcss/typography**: Purpose: Official Tailwind CSS plugin that adds `prose` classes for styling long-form HTML content such as Markdown or CMS output. Provides typographic defaults and modifiers for elements like headings, links, lists, and code blocks.
- **@tanstack/electric-db-collection**: Purpose: Provides `electricCollectionOptions` for TanStack DB collections that sync Postgres data through ElectricSQL shapes. It supports real-time, local-first apps with optimistic updates and custom mutation handlers.
- **@tanstack/query-db-collection**: Purpose: TanStack DB integration for TanStack Query. It provides `queryCollectionOptions` to build query-backed collections that fetch remote data, support optimistic updates with rollback, and use customizable mutation handlers for sync/persistence. Sources: https://tanstack.com/db/latest/docs/collections/query-collection, https://tanstack.com/db/latest/docs/reference/query-db-collection/index
- **@tanstack/react-db**: Purpose: React adapter for TanStack DB. It re-exports the core DB APIs and adds React hooks for live queries, suspense queries, and paced mutations.
- **@tanstack/react-form**: Purpose: React form library for building large-scale, production-ready forms with a focus on type safety, performance, and composition. It provides hooks and helpers such as useForm, useField, and createFormHook for typed form state, validation, and reusable form components.
- **@tanstack/react-query**: Purpose: TanStack Query is a React server-state and data-fetching library for fetching, caching, updating, and refetching async data with automatic background updates and no global state. It’s written in TypeScript and provides type-safe hooks and query/mutation helpers for React apps.
- **@tanstack/react-query-devtools**: Purpose: Provides React Query DevTools for debugging TanStack Query in React apps. It lets you visualize queries in a floating or embedded panel, and is excluded from production bundles by default.
- **@tanstack/react-router**: Purpose: TanStack Router for React: a type-safe routing library for React applications. It supports route trees, loaders/data fetching, search params, and integration via RouterProvider.
- **@tanstack/react-table**: Purpose: The `@tanstack/react-table` adapter is a wrapper around the core table logic. It handles React-style state management and provides types plus rendering for cell, header, and footer templates.
- **@tanstack/react-virtual**: Purpose: React adapter for TanStack Virtual. It wraps the core virtualizer logic and provides hooks like useVirtualizer and useWindowVirtualizer for rendering large, scrollable lists efficiently in React.
- **@tanstack/zod-adapter**: Purpose: TanStack Router adapter for Zod schemas, providing `zodValidator` and `fallback` helpers so search params can be validated with correct TypeScript input/output types. Official docs note it’s mainly needed for Zod v3; Zod v4 can use native Zod features directly.
- **@tanstack/zod-form-adapter**: Purpose: The Zod adapter for TanStack Form. It provides a `zodValidator`-style validator adapter so TanStack Form can validate form and field values with Zod schemas.
- **axios**: Purpose: A promise-based HTTP client for the browser and Node.js, with a simple API for making requests and handling interceptors. It declares types and has full TypeScript support.
- **canvas-confetti**: Purpose: performant confetti animation in the browser. Provides a `confetti([options])` function for launching browser confetti with options like particle count, spread, colors, shapes, and reduced-motion support.
- **class-variance-authority**: Purpose: Type-safe utility for composing CSS class variants; provides a `cva` function plus `VariantProps` helper to build variant-based component APIs in TypeScript.
- **clsx**: Purpose: A tiny utility for constructing `className` strings conditionally. It accepts strings, arrays, objects, booleans, and other mixed inputs, discarding falsey values; it’s also a smaller drop-in replacement for `classnames`.
- **cmdk**: Purpose: A React command menu component that can also be used as an accessible combobox. It renders items and automatically filters/sorts them, with a fully composable API.
- **date-fns**: Purpose: A modern JavaScript/TypeScript date utility library for manipulating dates in a browser and Node.js. It provides a comprehensive, consistent set of small, functional helpers for working with native JavaScript Date objects.
- **dnd kit**: Purpose: A modern toolkit for building drag-and-drop interfaces with first-class TypeScript support. It provides core drag-and-drop primitives, plus sortable and utility packages for reordering and customization.
- **flyonui**: Purpose: Open-source Tailwind CSS component library with 80+ prebuilt UI components, blocks, templates, and plugins. It uses semantic component classes and modifier classes for customization and interactive behavior.
- **json2md**: Purpose: Converts a JSON input into Markdown. It supports generating common Markdown elements like headings, paragraphs, blockquotes, images, lists, and code blocks from structured data.
- **lucide-react**: Purpose: React components for Lucide icons that render as optimized inline SVGs. The package is fully typed for TypeScript, tree-shakable, and lets you customize icon props like size, color, and stroke width.
- **motion**: Purpose: Motion is an open-source animation library for JavaScript, React, and Vue. It provides a simple API for animating CSS properties, SVG, and JavaScript values, and is fully TypeScript-typed.
- **nanoid**: Purpose: A tiny, secure, URL-friendly unique string ID generator for JavaScript. In TypeScript, it also supports casting generated IDs to branded/opaque string types.
- **next-themes**: Purpose: An abstraction for themes in your React app. It supports system preference, avoids flash on load, and provides a `ThemeProvider` plus `useTheme` for switching themes.
- **nuqs**: Purpose: React library for keeping state in sync with the URL query string. It offers a `useState`-style API with type-safe parsers/serializers and support for Next.js, Remix, React Router, TanStack Router, and plain React SPAs.
- **Radix UI React Primitives**: Purpose: An open-source React UI component library for building high-quality, accessible design systems and web apps. The primitives are low-level, unstyled, fully typed, and designed for incremental adoption with accessible behavior handled internally.
- **react**: Purpose: React is a library for building web and native user interfaces. It lets you compose UIs from reusable components and supports modern features like hooks and JSX.
- **react-day-picker**: Purpose: React component for creating date pickers, calendars, and date inputs for web applications. Supports single, multiple, and range selections, localization/time zones, and is written in TypeScript.
- **react-dom**: Purpose: Provides React’s web-only DOM renderer: entry points for client and server rendering in browser DOM applications, plus APIs like createRoot, hydrateRoot, createPortal, and flushSync.
- **react-icons**: Purpose: A React icon library that packages popular icon sets as ES6-importable SVG components, so you can import only the icons your app uses. It has native TypeScript support with built-in type declarations.
- **react-markdown**: Purpose: React component for rendering a string of Markdown into React elements, safely by default. Supports remark/rehype plugins and custom components to control how markdown is transformed and displayed.
- **shadcn-ui**: Purpose: A set of beautifully-designed, accessible components and a code distribution platform. It is not a traditional component library; it helps you build your own component library.
- **sonner**: Purpose: An opinionated toast component for React. Render a `<Toaster />` in your app and trigger toasts with `toast()`.
- **tailwind-merge**: Purpose: Utility for efficiently merging Tailwind CSS class strings in JavaScript/TypeScript without style conflicts. It helps resolve conflicting utility classes when composing styles, and its API is fully typed.
- **vaul**: Purpose: Vaul is a drawer component for React. It provides Drawer primitives such as Root, Trigger, Portal, Overlay, and Content for building drawers in your app.
- **wicg-inert**: Purpose: Polyfill for the `inert` attribute and property. It lets web authors mark parts of the DOM as inert so they don’t receive user interaction events and are hidden from assistive technology.
- **zod**: Purpose: TypeScript-first schema validation library with static type inference. It lets you define schemas to validate and parse data—from primitives to complex nested objects—while producing type-safe results.
- **zustand**: Purpose: A small, fast, scalable state-management library for React and vanilla stores. It uses a hook-based API and has first-class TypeScript support for typed state, actions, and selectors.
