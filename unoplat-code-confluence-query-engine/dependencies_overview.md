# Dependencies Overview

- **Package management**: Use `uv` as the package manager for this codebase.
- **aiofiles**: Purpose: Provides async-compatible file handling for asyncio applications, mirroring Python’s standard file API while delegating blocking file operations to a thread pool. Also includes async tempfile helpers and selected async os/path utilities.
- **aiopath**: Purpose: Async pathlib for Python. It is a complete, pathlib-compatible implementation whose I/O methods are awaitable and work with asyncio/trio, letting filesystem access stay non-blocking.
- **asyncpg**: Purpose: An asynchronous database interface library for PostgreSQL and Python/asyncio. It provides an efficient implementation of PostgreSQL’s binary protocol for use with Python’s asyncio framework.
- **fastapi**: Purpose: A modern, high-performance Python web framework for building APIs with standard Python type hints. It includes automatic interactive API documentation and OpenAPI-based validation/documentation.
- **ghapi**: Purpose: Python interface to GitHub’s API, with a Pythonic wrapper generated from GitHub’s OpenAPI spec and kept up to date with the latest REST endpoints. It can be used from Python or the command line to automate GitHub actions like issues, pull requests, releases, webhooks, and more.
- **jinja2**: Purpose: A fast, expressive, extensible templating engine for Python. Templates use Python-like syntax and support features like inheritance, macros, autoescaping, sandboxing, and async rendering.
- **logfire**: Purpose: Pydantic Logfire is an observability platform built on OpenTelemetry with native Python support. It lets you instrument applications and send logs, traces, and metrics to Logfire.
- **loguru**: Purpose: A pre-instanced logging library for Python that aims to make logging easy and pleasant to use. It provides a ready-to-use logger with features like formatting, rotation/retention/compression, colors, structured logging, and compatibility with standard logging.
- **markdown-it-py**: Purpose: A Python port of markdown-it: a Markdown parser that follows the CommonMark spec, with configurable syntax and plugin-based extensions. It renders Markdown to HTML and is designed for speed and security.
- **pydantic-ai**: Purpose: Python agent framework for building production-grade GenAI applications and workflows. Model-agnostic, with built-in support for many providers plus tools, structured outputs, and observability via Pydantic Logfire.
- **pydantic-ai-backend**: Purpose: Provides file storage and sandbox backends for Pydantic AI agents, with in-memory, local filesystem, and Docker-isolated execution options. Also includes a console toolset and permission controls for safe file operations and code execution.
- **pydantic-ai-skills**: Purpose: A standardized, composable framework for building and managing Agent Skills within the Pydantic AI ecosystem. It implements the Agent Skills specification with progressive disclosure, supporting filesystem-based and programmatic skills for on-demand loading.
- **pydantic-ai-slim**: Purpose: Slim install of Pydantic AI for building Python GenAI agents. It lets you install only the core framework plus optional extras for the specific model providers and features you need.
- **qs-codec**: Purpose: A query string encoding and decoding library for Python. A Python port of the JavaScript qs library, preserving its list/duplicate-handling and formatting behavior.
- **sqlalchemy**: Purpose: SQLAlchemy is the Python SQL toolkit and Object Relational Mapper for working with databases. Its Core provides SQL expression, DBAPI/transaction, and schema services; the ORM adds optional object-relational mapping and persistence.
- **sqlmodel**: Purpose: A Python library for working with SQL databases through Python objects and type-annotated models. It is built on Pydantic and SQLAlchemy, and is designed to work well with FastAPI.
- **sse-starlette**: Purpose: Production-ready Server-Sent Events (SSE) support for Starlette and FastAPI, implementing the W3C SSE specification. It provides `EventSourceResponse` and helpers for streaming events over HTTP.
