# Dependencies Overview

- **Package management**: Use `uv` as the package manager for this codebase.
- **aiofiles**: Purpose: Python library for asynchronous file I/O in asyncio applications. It provides async versions of standard file and tempfile operations, delegating blocking work to a thread pool.
- **aiopath**: Purpose: Async pathlib for Python: a complete implementation of `pathlib` compatible with `asyncio`, `trio`, and `async/await`. All I/O is asynchronous and awaitable, so filesystem operations can be used without blocking the event loop.
- **asyncpg**: Purpose: A database interface library designed specifically for PostgreSQL and Python/asyncio. It provides an efficient implementation of the PostgreSQL binary protocol, with features like connection pooling and automatic PostgreSQL↔Python type conversion.
- **fastapi**: Purpose: A modern, high-performance web framework for building APIs with Python based on standard Python type hints. Includes automatic interactive API docs (Swagger UI/ReDoc) and is built on Starlette and Pydantic.
- **ghapi**: Purpose: A Python client and command-line interface for GitHub’s REST API. It dynamically generates a Pythonic interface from GitHub’s OpenAPI spec, aiming to provide complete, always-updated coverage of the API.
- **logfire**: Purpose: Pydantic Logfire is an OpenTelemetry-based observability platform with a first-party Python SDK for logging, tracing, and metrics. It helps you instrument Python apps and popular libraries to monitor application behavior.
- **loguru**: Purpose: Python logging library that provides a pre-configured, easy-to-use logger with one `add()` API for sinks, formatting, filtering, rotation, and retention. It supports structured, async/thread-safe logging and integrates with standard `logging`.
- **pydantic-ai**: Purpose: A Python agent framework for building production-grade generative AI applications and workflows. It is model-agnostic and supports multiple LLM providers with built-in tools for agents, testing, observability, and durable execution.
- **pydantic-ai-skills**: Purpose: A standardized, composable framework for building and managing Agent Skills in the Pydantic AI ecosystem. It lets agents discover, load, and execute skills made up of instructions, scripts, tools, and resources using the Agent Skills specification.
- **pydantic-ai-slim**: Purpose: Slim installation of Pydantic AI that includes the core agent framework with minimal required dependencies. Optional extras can be installed for specific model providers and add-ons such as OpenAI, Google, Logfire, or Outlines.
- **qs-codec**: Purpose: Python query-string encoding and decoding library, ported from the qs package for JavaScript/Node.js. It provides encode/decode helpers and options for list formats, duplicate-key handling, charset sentinels, and RFC 3986/RFC 1738 formatting.
- **sqlalchemy**: Purpose: SQLAlchemy is a Python SQL toolkit and Object Relational Mapper for working with databases. It provides Core SQL/expression-language tools plus an ORM for mapping Python objects to database schemas.
- **sqlmodel**: Purpose: SQLModel is a Python library that combines SQLAlchemy and Pydantic so you can define database models with standard type annotations. It is designed to make working with SQL databases intuitive, with FastAPI-friendly models and automatic validation/serialization from Pydantic.
- **sse-starlette**: Purpose: Production-ready Server-Sent Events support for Starlette and FastAPI following the W3C SSE spec. Provides EventSourceResponse and helper event types for streaming SSE-compliant responses over ASGI.
