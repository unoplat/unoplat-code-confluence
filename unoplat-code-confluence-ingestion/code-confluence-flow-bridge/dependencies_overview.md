# Dependencies Overview

- **Package management**: Use `uv` as the package manager for this codebase.
- **aiofile**: Purpose: Real asynchronous file operations with asyncio support. Provides a high-level async_open file-like interface plus low-level AIOFile/Reader/Writer/LineReader helpers for asynchronous file I/O.
- **asyncpg**: Purpose: A database interface library designed specifically for PostgreSQL and Python/asyncio. It provides a native implementation of the PostgreSQL server binary protocol for async Python apps.
- **fastapi**: Purpose: FastAPI is a modern, high-performance web framework for building APIs with Python based on standard Python type hints. It also provides automatic interactive API documentation (Swagger UI/ReDoc) and is designed to be easy to use, fast to code, and production-ready.
- **gitpython**: Purpose: Python library for working with Git repositories at both high-level porcelain and low-level plumbing. It provides object abstractions for repository data and can operate via pure Python or the Git command-line backend.
- **gql**: Purpose: GQL is a GraphQL client for Python that works with GraphQL implementations compatible with the spec. It uses GraphQL-core under the hood and supports multiple transports for sync and async usage.
- **greenlet**: Purpose: Lightweight coroutines for in-process concurrent programming, providing cooperatively scheduled greenlets that switch explicitly under programmer control. Often used as a low-level building block for frameworks like gevent and other custom control-flow structures.
- **grpcio**: Purpose: Package for gRPC Python. It provides the Python runtime for building clients and servers with gRPC, an HTTP/2-based RPC framework.
- **loguru**: Purpose: Loguru is a Python logging library that provides a pre-configured, ready-to-use logger with a simpler API than the standard library. It supports features like rotation/retention, colorized output, structured logging, and is fully compatible with Python’s standard logging.
- **opentelemetry-api**: Purpose: Provides the OpenTelemetry Python API for instrumenting applications and producing telemetry data. Used with `opentelemetry-sdk` to generate and collect traces, metrics, and logs.
- **opentelemetry-exporter-otlp**: Purpose: Allows exporting telemetry data to the OpenTelemetry Collector using the OpenTelemetry Protocol (OTLP).
- **opentelemetry-instrumentation-logging**: Purpose: Automatically instruments Python's stdlib logging with a handler that converts log messages into OpenTelemetry logs and exports them. It can also inject trace context into log records and optionally set a logging format for correlation.
- **opentelemetry-sdk**: Purpose: Reference implementation of the OpenTelemetry API for Python. Used by applications to generate and collect telemetry data such as traces, metrics, and logs.
- **packaging**: Purpose: Reusable core utilities for Python packaging interoperability specifications. Implements packaging specs with one correct behavior or shared implementations, such as version handling (PEP 440) and compatibility tags (PEP 425).
- **passlib**: Purpose: Passlib is a Python password-hashing library for managing password hashes, password contexts, two-factor authentication, and related utilities. It provides cross-platform implementations of 30+ hashing algorithms and a uniform API for hashing and verifying passwords.
- **psycopg2-binary**: Purpose: Stand-alone Psycopg 2 package for Python/PostgreSQL: a pre-compiled binary distribution of the psycopg2 module that does not require a compiler or external libraries. Official docs: https://www.psycopg.org/docs/install.html ; PyPI: https://pypi.org/project/psycopg2-binary/
- **pydantic**: Purpose: Data validation using Python type annotations; parse and validate complex data, generate JSON schemas, and ensure data integrity. Pydantic is the most widely used data validation library for Python.
- **pydantic-settings**: Purpose: Provides optional Pydantic features for loading a settings or config class from environment variables or secrets files. It exposes `BaseSettings` for building typed application settings and customizing how settings sources are loaded.
- **pygithub**: Purpose: PyGithub is a Python library for accessing the GitHub REST API v3. It lets you manage GitHub resources such as repositories, users, and organizations from Python code.
- **pyyaml**: Purpose: A full-featured YAML framework for Python: it parses YAML into Python objects and emits Python objects back to YAML. It also offers safe loading/dumping APIs and optional LibYAML-based speedups.
- **requirements-parser**: Purpose: A Python module for parsing pip requirement files. It handles requirement specifiers, version-control URLs, other URIs, and local file paths.
- **sqlalchemy**: Purpose: SQLAlchemy is a comprehensive SQL toolkit and Object Relational Mapper for working with databases and Python. It provides Core SQL expression tools, ORM mapping, engine/connection/pooling services, and database dialect support.
- **sqlmodel**: Purpose: A Python library for interacting with SQL databases using Python objects. It is built on Python type annotations and powered by Pydantic and SQLAlchemy, aiming to be simple, intuitive, and reduce code duplication.
- **temporalio**: Purpose: Python SDK for Temporal. It lets you build and run workflows and activities in Python, with a client for connecting to Temporal and a worker for processing workflows/activities.
- **tiktoken**: Purpose: A fast BPE tokenizer for use with OpenAI’s models. It provides encoding/decoding utilities and model-specific encodings such as `encoding_for_model`.
- **tomlkit**: Purpose: Style-preserving TOML library for Python. It parses and writes TOML 1.1.0 while preserving comments, whitespace, and element ordering through an editable API.
- **tree-sitter-language-pack**: Purpose: Bundles 306 tree-sitter grammars behind a single Rust core with native Python bindings. Parsers download on demand and cache locally, giving one API for parsing, code intelligence, and syntax-aware chunking.
- **typing-extensions**: Purpose: Backported and experimental type hints for Python. It lets you use newer typing features on older Python versions and try new typing PEPs before they are added to the standard `typing` module.
- **validate-pyproject**: Purpose: Command-line tool and Python library for validating `pyproject.toml` files using JSON Schema. Includes checks for packaging-related standards such as PEP 517, PEP 518, PEP 621, PEP 639, and PEP 735.
