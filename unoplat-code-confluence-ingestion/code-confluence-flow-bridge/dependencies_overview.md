# Dependencies Overview

- **Package management**: Use `uv` as the package manager for this codebase.
- **aiofile**: Purpose: Real asynchronous file operations with asyncio support. It provides async_open plus low-level AIOFile/Reader/Writer/LineReader helpers for async file I/O, including concurrent offset-based reads and writes.
- **asyncpg**: Purpose: An asynchronous PostgreSQL database interface for Python’s asyncio. It natively implements the PostgreSQL server binary protocol and supports efficient querying, transactions, connection pools, and type conversion.
- **fastapi**: Purpose: FastAPI is a modern, high-performance web framework for building APIs with Python and standard type hints. It provides automatic validation plus interactive API documentation based on OpenAPI (Swagger UI and ReDoc).
- **gitpython**: Purpose: GitPython is a Python library used to interact with Git repositories, with both high-level (porcelain) and low-level (plumbing) access. It provides abstractions for Git objects and repository data, backed by the git command-line program.
- **gql**: Purpose: GQL is a GraphQL client for Python that works with GraphQL implementations compatible with the spec. It supports sync and async usage, multiple transports, schema validation, subscriptions, and a CLI.
- **greenlet**: Purpose: Greenlets are lightweight coroutines for in-process concurrent programming. They let you create small pseudo-threads and explicitly switch execution between them within a single native thread, often as a foundation for frameworks like gevent.
- **grpcio**: Purpose: Package for gRPC Python; provides the Python runtime for building gRPC clients and servers from `.proto` service definitions.
- **loguru**: Purpose: Loguru is a Python logging library designed to make logging simpler and more pleasant to use. It provides a ready-to-use logger and extra features that reduce the boilerplate of the standard logging module.
- **opentelemetry-api**: Purpose: Python package for the OpenTelemetry API. It provides the interfaces used to instrument code and create telemetry signals; applications typically pair it with the SDK, while instrumented libraries can depend on the API alone.
- **opentelemetry-exporter-otlp**: Purpose: Convenience package for OpenTelemetry Python that installs the OTLP exporter implementations. It provides exporters for sending telemetry to an OpenTelemetry Collector over gRPC or HTTP/protobuf.
- **opentelemetry-instrumentation-logging**: Purpose: Automatically instruments Python’s stdlib logging with a handler that converts log messages into OpenTelemetry logs and exports them. It can also inject trace context into log records and optionally set a correlated log format via `LoggingInstrumentor`.
- **opentelemetry-sdk**: Purpose: Reference implementation of the OpenTelemetry API for Python. Use it in applications to generate and collect telemetry data such as traces, metrics, and logs.
- **packaging**: Purpose: Reusable core utilities for Python Packaging interoperability specifications. It provides utilities for standards that need one shared implementation, such as PEP 440 version handling and related packaging metadata/specifier logic.
- **passlib**: Purpose: Passlib is a collection of routines for managing password hashes such as those found in Unix shadow files, `crypt.crypt()`, MySQL, and PostgreSQL. It also provides password contexts, two-factor authentication helpers, and other utility functions.
- **psycopg2-binary**: Purpose: Pre-compiled binary distribution of Psycopg 2, the PostgreSQL adapter for Python. It installs without build/runtime prerequisites; you still import it as `psycopg2`.
- **pydantic**: Purpose: Python data validation library that uses type annotations to define models, parse and validate data, and generate JSON Schema. It also supports serialization and other model utilities via BaseModel.
- **pydantic-settings**: Purpose: Pydantic Settings provides optional Pydantic features for loading a settings or config class from environment variables, dotenv files, and secrets files. It also includes integrated CLI support for using Pydantic models as command-line apps.
- **pygithub**: Purpose: PyGithub is a Python library for using the GitHub API v3. It lets you manage GitHub resources such as repositories, user profiles, and organizations from Python scripts.
- **pyyaml**: Purpose: A full-featured YAML framework for the Python programming language. It provides YAML parsing and emitting, plus APIs for serializing/deserializing native Python objects.
- **requirements-parser**: Purpose: Python module for parsing Pip requirement files. It handles requirement specifiers, version control URIs, local file paths, and other requirement-file forms.
- **sqlalchemy**: Purpose: Python database toolkit and ORM for working with databases. It provides a Core SQL expression language plus an object-relational mapper for schema-centric or object-centric database access.
- **sqlmodel**: Purpose: SQLModel is a Python library for working with SQL databases using standard type annotations. It combines Pydantic-style data validation with SQLAlchemy ORM models, and can be used with FastAPI or any other Python application.
- **temporalio**: Purpose: Python SDK for Temporal, used to build Temporal applications. It provides client and worker APIs for running workflows and activities.
- **tiktoken**: Purpose: A fast BPE tokeniser for use with OpenAI’s models. It provides model-specific encodings and APIs like `get_encoding()` and `encoding_for_model()` for converting text to tokens.
- **tomlkit**: Purpose: Style-preserving TOML library for Python. It provides a 1.0.0-compliant TOML parser/writer that preserves comments, indentation, whitespace, and key ordering, and lets you create and edit TOML documents through an API.
- **tree-sitter-language-pack**: Purpose: Python bindings for tree-sitter-language-pack, providing access to 305 pre-compiled tree-sitter parsers with on-demand downloads. Also exposes parser and `process()` APIs for parsing source code and extracting structured analysis.
- **typing-extensions**: Purpose: Backports and exposes newer Python type-hinting features on older Python versions, while also providing experimental typing features before they land in the standard library `typing` module. It complements the standard-library `typing` module and is used by static type checkers like mypy and pyright.
- **validate-pyproject**: Purpose: Command-line tool and Python library for validating `pyproject.toml` files with JSON Schema. It includes checks for packaging standards such as PEP 517, PEP 518, PEP 621, PEP 639, and PEP 735.
